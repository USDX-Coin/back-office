# USDX Back-Office

## Overview

Internal back-office admin panel untuk operasional USDX stablecoin. Manage users (staf & client), OTC mint/burn, dan reporting. Phase 1: frontend only, semua API di-mock via MSW.

**Docs referensi:**
- `USDX-Knowledge.md` ‚Äî feature spec & layout convention
- `USDX-Foundation-Decisions.md` ‚Äî locked technology decisions
- `USDX-Backoffice-Scaffolding-Spec.md` ‚Äî scaffolding spec

**Agent instructions:** Baca `docs/brief/USDX-Agent-Instructions.md` untuk communication style dan workflow rules.

## Tech Stack

| Category | Choice |
|---|---|
| Build tool | Vite 8 |
| Language | TypeScript (strict) |
| Package manager | **pnpm** |
| Styling | TailwindCSS v4 (CSS-first, `@tailwindcss/vite`) |
| Components | shadcn/ui (Radix UI primitives) |
| Routing | TanStack Router v1 (file-based) |
| Data fetching | TanStack React Query v5 |
| Form | react-hook-form + zod |
| Table | TanStack Table v8 |
| State | Zustand (+ persist middleware) |
| Mock backend | MSW v2 (worker mode) |
| Date | date-fns v4 + date-fns-tz (locale `id`, WIB hardcoded) |

## Conventions

### Naming
- Files: `kebab-case.ts` / `kebab-case.tsx`
- Components: `PascalCase` export
- Hooks: `useCamelCase`
- Types: `PascalCase`
- Zod schemas: `camelCaseSchema`
- Path alias: `@/*` ‚Üí `src/*` (selalu pakai, jangan relative)

### TanStack Router (TSR) ‚Äî Aditya belum familiar
- `__root.tsx` (dua underscore) = root, parent semua
- `_app.tsx` (satu underscore) = pathless layout, `_app` tidak masuk URL
- `_app.<segment>.tsx` = child route dengan shell, URL = `/<segment>`
- `routeTree.gen.ts` = auto-generated, jangan edit manual
- Auth guard via `beforeLoad`: `throw redirect({ to: '/login' })`
- Type-safe navigation: `<Link to="..." />`, `navigate({ to: '...' })`

### Sidebar Nav (locked)

Dashboard    ‚Üí /

  Staf       ‚Üí /users-staf

### Data Flow
```
Route ‚Üí useQuery hook ‚Üí apiFetch() ‚Üí MSW handler (dev) / Real API (prod)
Route ‚Üí useMutation hook ‚Üí apiFetch() ‚Üí MSW handler
  ‚îî‚Üí onSuccess: queryClient.invalidateQueries()
```

### API Contract (frontend-led)
```ts
// List:    { data: T[], meta: { page, pageSize, total, totalPages } }
// Single:  { data: T }
// Error:   { error: { code: string, message: string } }
```

### Color System (USDX Brand)
- Primary: `#1eaed5` ‚âà HSL `190 75% 48%` ‚Äî cyan-teal
- CSS var: `--primary: 190 75% 48%`
- Tailwind: `bg-primary`, `text-primary-foreground`

### MSW Handlers Pattern
- Delay 200-800ms (realistic)
- Success: `HttpResponse.json({ data: T })`
- Error: `HttpResponse.json({ error: { code, message } }, { status: 4xx })`

## Adding New Feature

1. Create `src/features/{name}/` ‚Äî components, api.ts, schema.ts, types.ts
2. Route file sudah ada di `src/routes/_app.{name}.tsx` ‚Äî replace placeholder
3. Add MSW handlers di `src/mocks/handlers/{name}.ts`, register di `handlers/index.ts`
4. Add mock data di `src/mocks/db.ts`
