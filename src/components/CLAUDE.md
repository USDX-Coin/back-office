# Components

Shared, reusable components used across multiple features.

## Structure

```
components/
  ├── ui/                    # shadcn/ui primitives — DO NOT EDIT MANUALLY
  ├── layout/                # App shell: AppSidebar, Navbar, MainLayout, ProfileDropdown, AuthGuard
  ├── Avatar.tsx             # Initials + fixed palette
  ├── StatusPill.tsx         # Single component for status / role / type pills
  ├── TableEmptyState.tsx    # Table empty-state with no-data / no-results modes
  ├── CustomerTypeahead.tsx  # Shared customer lookup (debounced)
  ├── DataTable.tsx          # Generic v2 data table — see DataTable Props below
  ├── DateRangePicker.tsx    # Calendar + preset side-panel range picker
  ├── PageHeader.tsx         # Minimal title + actions slot
  ├── SummaryStat.tsx        # Lightweight KPI card (still available, not used by current pages)
  └── useDataTableParams.ts  # Per-page URL filter helper (separate from DataTable's nuqs binding)
```

## Rules

- **`ui/`** — Generated by `npx shadcn@latest add <component>`. Do not modify these files directly. Customizations go in consuming components or via Tailwind theme tokens in `src/index.css`. **Tailwind v4 caveat**: arbitrary class values that read CSS variables must wrap them in `var()`, e.g. `w-[var(--sidebar-width)]` not `w-[--sidebar-width]`. Patch the generated file when the CLI emits the v3 form.
- **`layout/`** — App-wide layout components. `MainLayout` renders `<SidebarProvider>` + `<AppSidebar />` + `<SidebarInset>(Navbar + <Outlet />)`. `AppSidebar` is composed from the shadcn Sidebar block; its menu sections live in `nav-config.ts`. `Navbar` is intentionally minimal: SidebarTrigger + breadcrumb + ProfileDropdown.
- **`Avatar`** — Initials only. Background hue indexed by `name.charCodeAt(0) % 8` from a fixed neutral palette.
- **`StatusPill`** — Wraps the status / role / type config in `src/lib/status.ts`. Three appearances (`filled`, `soft`, `outline`) and five tones (`success`, `warning`, `destructive`, `info`, `neutral`). Pass `status="…"` to auto-resolve OTC status, or `label + tone` for non-OTC pills.
- **`TableEmptyState`** — Two modes: `no-data` (default empty surface, optional CTA) and `no-results` (filtered with optional Clear-filters affordance).
- **`CustomerTypeahead`** — Debounced (300 ms) input backed by `useCustomers({search})`. Handles idle / debounce / in-flight (skeletons) / results / zero-results / error states. Used by Users, OTC Mint, OTC Redeem, Report.
- **`DataTable`** — TanStack Table v8 with column resizing, column visibility toggle, row selection, page-size selector, server-driven nuqs URL state, and a slot for per-page filter toolbars. See "DataTable v2" below.
- **`DateRangePicker`** — `react-day-picker` Calendar in range mode inside a Popover, with a preset side panel (Last hour / 7 / 14 / 30 days) and Custom Range datetime inputs. Emits ISO `yyyy-MM-dd` strings.
- **`PageHeader`** — Renders an `<h1>` plus an optional actions slot. No eyebrow, italic accent, or subtitle — pages are intentionally "title, then table."
- **`useDataTableParams`** — Plain `useSearchParams` wrapper for pages that maintain per-page URL state (type/role/status filters) outside the table's own page/pageSize/sort/search keys. Returns `searchParams`, `page`, `sortBy`, `sortOrder`, `search`, plus `updateParams` and `clearAll`.

## DataTable v2

Generic, server-driven table component on TanStack Table v8.

### Props

| Prop | Type | Description |
|------|------|-------------|
| `columns` | `ColumnDef<T>[]` | TanStack Table column definitions; supports `meta.filterType`, `meta.align`, `meta.headerClassName`, `meta.cellClassName` |
| `data` | `T[]` | Current page data |
| `rowCount` | `number` | Total row count for pagination |
| `isLoading` | `boolean` | Shows skeleton rows |
| `enableRowSelection` | `boolean` | Enables the toolbar's "Select Rows" toggle |
| `onRowSelectionChange` | `(state) => void` | Called when row selection changes |
| `disableColumnResize` | `boolean` | Hides resize handles |
| `filterToolbar` | `ReactNode` | Custom toolbar; replaces the default search+status+date toolbar when provided |
| `emptyState` | `ReactNode` | Custom no-data slot |
| `hasFilters` | `boolean` | Override; when true and rowCount=0, renders the no-results empty state |
| `statusOptions` | `{value, label}[]` | Default-toolbar status select options |
| `onExportCsv` | `() => void` | Default-toolbar CSV export callback |
| `pageSize` | `number` | Default 10 |
| `onRowClick` | `(row) => void` | Renders rows as `cursor-pointer` and calls back on click |

### URL state

Internal state bound to nuqs parsers from `src/lib/url-state.ts`:

| Key | Parser | Default |
|-----|--------|---------|
| `page` | integer | 1 |
| `pageSize` | integer | 10 |
| `search` | string | empty |
| `sortBy` | string | empty |
| `sortOrder` | enum `asc | desc` | desc |

URL key names are part of the contract with MSW handlers — do not rename.
